# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект демонстрирует применение методов и инструментов инженерии ИИ: работу с данными, модели, пайплайны, сервис, эксперименты и воспроизводимость.

---

## 1. Паспорт проекта

- **Название проекта:** Сервис кредитного скоринга: предсказание вероятности дефолта заёмщика
- **Автор:** Белов Владимир Андреевич
- **Группа:** КМБО-11-24
- **Контакт:** `@fimaruyn`

- **Краткое описание (2-4 предложения):**  
  Проект реализует end-to-end сервис для оценки кредитного риска на основе табличных данных. Задача бинарной классификации решается с помощью градиентного бустинга (LightGBM), обученного на открытом датасете Give Me Some Credit. Результат — контейнеризованный REST API на FastAPI, который по анкетным данным заёмщика возвращает вероятность дефолта, категорию риска и решение о выдаче кредита. Архитектура включает модульный пайплайн предобработки, базовую наблюдаемость и безопасное управление конфигурацией.

---

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - `01_eda.ipynb` – разведочный анализ, визуализации, гипотезы предобработки.
  - `02_modeling.ipynb` – обучение, сравнение моделей, оценка метрик, сохранение артефактов.
- `src/` – основной код проекта:
  - `src/data/` – загрузка и валидация данных (pandera схемы, логирование).
  - `src/features/` – пайплайн предобработки (импутация, трансформации, масштабирование).
  - `src/service/` – FastAPI приложение, Pydantic схемы, конфигурация, middleware.
- `data/` – демонстрационные/учебные данные:
  - `raw/` – исходные CSV-файлы (не коммитятся в репозиторий из-за объёма).
  - `README.md` – инструкция по загрузке датасета.
- `configs/` – конфигурационные файлы:
  - `.env.example` – шаблон переменных окружения.
  - `config.py` – типизированные настройки через `pydantic-settings`.
- `tests/` – тесты (юнит-тесты валидации данных, интеграционные тесты API).
- `artifacts/` – сохранённые модели, отчёты, визуализации и метаданные экспериментов.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.12`
- `uv` (рекомендуемый менеджер пакетов и виртуальных окружений)
- Docker Engine `>= 20.10` (для контейнеризации)
- Git

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Создать и активировать виртуальное окружение через uv
uv venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate   # Windows

# Установить зависимости
uv pip install -r requirements.txt

# Скопировать шаблон конфигурации (обязательно перед запуском)
cp configs/.env.example configs/.env
```

---

## 4. Как запустить проект

### 4.1. Локальный запуск сервиса (API)

```bash
cd project
source .venv/bin/activate
uv run uvicorn src.service.main:app --host 0.0.0.0 --port 8000 --reload
```

Сервис поднимается на `http://localhost:8000`.  
Доступна интерактивная документация Swagger UI: `http://localhost:8000/docs`

### 4.2. Запуск в Docker

```bash
cd project
docker build -t credit-scorer .
docker run -d --name scorer-service -p 8000:8000 --env-file configs/.env credit-scorer
```

### 4.3. Ключевые эндпоинты и тестирование

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/health` | Проверка статуса сервиса и загрузки модели |
| `POST` | `/predict` | Предсказание кредитного скоринга |

**Пример запроса:**
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "RevolvingUtilizationOfUnsecuredLines": 0.15,
    "age": 35,
    "NumberOfTime30-59DaysPastDueNotWorse": 0,
    "DebtRatio": 0.3,
    "MonthlyIncome": 5000,
    "NumberOfOpenCreditLinesAndLoans": 3,
    "NumberOfTimes90DaysLate": 0,
    "NumberRealEstateLoansOrLines": 1,
    "NumberOfTime60-89DaysPastDueNotWorse": 0,
    "NumberOfDependents": 2
  }'
```

**Ожидаемый ответ:**
```json
{
  "default_probability": 0.0842,
  "risk_category": "low",
  "decision": "approve",
  "model_version": "1.0.0",
  "request_id": "uuid-v4-string"
}
```

## 5. Данные

- **Источник**: Открытый датасет [Give Me Some Credit (Kaggle)](https://www.kaggle.com/c/GiveMeSomeCredit)
- **Объём**: ~150 000 записей, 10 числовых признаков + 1 целевая переменная (`SeriousDlqin2yrs`)
- **Хранение**: 
  - Исходный файл `cs-training.csv` размещается в `data/raw/` локально. 
  - Файл **не коммитится** в репозиторий (правило `faq.md §2.3`). Инструкция по загрузке находится в `data/README.md`.
- **Предобработка**: реализована в `src/features/preprocessing.py` через `sklearn.pipeline`. Включает импутацию пропусков, логарифмическую трансформацию skewed-признаков, агрегацию коррелирующих счётчиков просрочек и масштабирование `RobustScaler`.

5. **Безопасность и конфигурация**: демонстрация `.env.example`, отсутствие секретов в репозитории, структура `src/`.

## 8. Ограничения и дальнейшая работа

**Текущие ограничения:**
- Статический порог принятия решений (фиксированные диапазоны вероятностей).
- Отсутствие мониторинга дрейфа данных (data drift) и онлайн-обучения.
- Базовая кросс-валидация без детального поиска гиперпараметров.

**Направления развития:**
- Интеграция SHAP для локальной интерпретации предсказаний (`/explain` endpoint).
- Добавление Prometheus-метрик (`/metrics`) и алертов при деградации качества.
- Автоматический пайплайн переобучения (MLflow/Airflow) и A/B-тестирование версий моделей.
- Калибровка вероятностей (`CalibratedClassifierCV`) для точной настройки бизнес-порогов.
