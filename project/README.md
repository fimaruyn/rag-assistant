# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект демонстрирует применение методов и инструментов инженерии ИИ: работу с данными, модели, пайплайны, сервис, эксперименты и воспроизводимость.

---

## 1. Паспорт проекта

- **Название проекта:** Сервис кредитного скоринга: предсказание вероятности дефолта заёмщика
- **Автор:** Белов Владимир Андреевич
- **Группа:** КМБО-11-24
- **Контакт:** `@fimaruyn`

- **Краткое описание (2-4 предложения):**  
  Проект реализует end-to-end сервис для оценки кредитного риска на основе табличных данных. Задача бинарной классификации решается с помощью градиентного бустинга (LightGBM), обученного на открытом датасете Give Me Some Credit. Результат — контейнеризованный REST API на FastAPI, который по анкетным данным заёмщика возвращает вероятность дефолта, категорию риска и решение о выдаче кредита. Архитектура включает модульный пайплайн предобработки, базовую наблюдаемость и безопасное управление конфигурацией.

## 3. Требования и установка

### 3.1. Системные требования

| Компонент | Минимальная версия | Примечание |
|-----------|-------------------|------------|
| **Python** | 3.12 | Рекомендуется через `uv` или `pyenv` |
| **uv** | 0.4+ | Менеджер пакетов и виртуальных окружений (опционально, но рекомендуется) |
| **Docker** | 20.10+ | Для контейнеризации (опционально, но рекомендуется) |
| **Git** | 2.30+ | Для работы с репозиторием |

### 3.2. Установка окружения (локально)

```powershell
# 1. Перейти в папку проекта
cd project

# (Опционально) Увеличить таймаут сетевых запросов uv
# $env:UV_HTTP_TIMEOUT = "300"

# 2. Создать и активировать виртуальное окружение через uv
uv venv .venv --python 3.12
.\.venv\Scripts\activate  # Windows PowerShell
# source .venv/bin/activate  # Linux/macOS

# 3. Установить зависимости
uv pip install -r requirements.txt

# 4. Инициализировать конфигурацию (обязательно перед запуском)
Copy-Item configs\.env.example configs\.env
# cp configs/.env.example configs/.env  # Linux/macOS
```

> **Примечание**: Файл `configs/.env` не коммитится в репозиторий. Все реальные значения хранятся локально.

## 4. Как запустить проект

### 4.1. Локальный запуск сервиса 

**Для разработки** (с горячей перезагрузкой, только папка `src/`):
```powershell
uv run uvicorn src.service.main:app --host 0.0.0.0 --port 8000 --reload --reload-dir src
```

- Сервис доступен по адресу: `http://localhost:8000`
- Swagger UI (интерактивная документация): `http://localhost:8000/docs`
- Health-check: `http://localhost:8000/health`

### 4.2. Запуск в Docker 

```powershell
# 1. Сборка образа
docker build -t credit-scorer .

# 2. Запуск контейнера
docker run -d --name scorer-service -p 8000:8000 --env-file configs/.env credit-scorer

# 3. Просмотр логов
docker logs -f scorer-service

# 4. Остановка и удаление
docker stop scorer-service && docker rm scorer-service
```

> **Важно**: Убедитесь, что файл `artifacts/modeling/models/scorer_pipeline_v1.0.joblib` присутствует в репозитории или скопирован в образ.

### 4.3. Запуск тестов

```powershell
# Все тесты с отчётом о покрытии
uv run pytest tests/ -v --cov=src --cov-report=html

# Только интеграционные тесты API
uv run pytest tests/test_api.py -v

# Только юнит-тесты данных
uv run pytest tests/test_data_loading.py -v
```

## 6. Данные

### 6.1. Источник и характеристики

| Параметр | Значение |
|----------|----------|
| **Датасет** | [Give Me Some Credit (Kaggle)](https://www.kaggle.com/c/GiveMeSomeCredit) |
| **Тип** | Открытый, обезличенный, учебный |
| **Объём** | ~150 000 записей |
| **Признаки** | 10 числовых (финансовые показатели, история просрочек, демография) |
| **Целевая переменная** | `SeriousDlqin2yrs` in {0, 1} (серьёзная просрочка 90+ дней в течение 2 лет) |
| **Лицензия** | Разрешено использование в некоммерческих и образовательных целях |

### 6.2. Загрузка данных

Исходные CSV-файлы **не коммитятся** в репозиторий. Для локального запуска:

1. Скачайте `cs-training.csv` со страницы датасета на Kaggle.
2. Поместите файл в `data/raw/`.
3. Убедитесь, что файл добавлен в `.gitignore` (по умолчанию уже добавлен).

```powershell
# Проверка наличия файла
Test-Path data\raw\cs-training.csv
# Ожидаемо: True (для локального запуска ноутбуков)
```

> **Примечание**: Сервис (`src/service/`) не требует сырых данных — он использует только сериализованный пайплайн из `artifacts/modeling/models/`.

### 6.3. Предобработка

Реализована в `src/features/preprocessing.py` через `sklearn.pipeline`:

| Этап | Метод | Обоснование |
|------|-------|-------------|
| **Фильтрация** | Удаление записей с `age < 18` или `age > 120` | Бизнес-правило валидности заёмщика |
| **Импутация** | Медиана для `MonthlyIncome`, константа 0 для `NumberOfDependents` | Устойчивость к выбросам, низкая доля пропусков |
| **Трансформация** | `log1p` для `RevolvingUtilization`, `DebtRatio`, `MonthlyIncome` | Стабилизация распределения с экстремальным перекосом (skewness > 90) |
| **Агрегация** | `TotalPastDue = sum(NumberOfTime*-PastDue*)` | Устранение мультиколлинеарности (|r| > 0.97) |
| **Масштабирование** | `RobustScaler` | Устойчивость к оставшимся выбросам |

## 7. Модели и эксперименты

### 7.1. Сравнение моделей

| Модель | ROC-AUC | Gini | Avg Precision | Время обучения |
|--------|---------|------|---------------|----------------|
| **LightGBM** (финальная) | **0.809** | **0.618** | **0.262** | 0.43 с |
| LogisticRegression | 0.794 | 0.588 | 0.217 | 0.19 с |
| RandomForest | 0.766 | 0.531 | 0.202 | 4.69 с |

### 7.2. Обоснование выбора LightGBM

1. **Качество**: ROC-AUC = 0.809 превышает индустриальный бенчмарк 0.74 для кредитного скоринга.
2. **Производительность**: обучение за 0.43 с, инференс < 50 мс — приемлемо для real-time API.
3. **Устойчивость**: автоматическая обработка пропусков и выбросов, не требует тщательного масштабирования.
4. **Интерпретируемость**: возможность извлечения feature importance для отчётности.

Подробное обоснование — в `report.md`, раздел «Выбор финальной модели».

## 9. Ограничения и дальнейшая работа

### 9.1. Текущие ограничения

| Ограничение | Влияние | Обходное решение |
|-------------|---------|------------------|
| Статический порог принятия решений | Фиксированные диапазоны вероятностей (0.10, 0.35) | Калибровка вероятностей через `CalibratedClassifierCV` |
| Отсутствие мониторинга дрейфа данных | Риск деградации качества со временем | Интеграция Evidently AI или Prometheus-алертов |
| Базовая кросс-валидация | Нет детального поиска гиперпараметров | Интеграция Optuna или GridSearchCV |
| Отсутствие аутентификации в API | Сервис открыт для всех | Добавление JWT-токенов или API-ключей |

### 9.2. Направления развития

1. **Интерпретируемость**: Интеграция SHAP для локальной интерпретации предсказаний (`/explain` endpoint).
2. **Мониторинг**: Добавление Prometheus-метрик (`/metrics`) и алертов при деградации качества.
3. **MLOps**: Автоматический пайплайн переобучения через MLflow/Airflow и A/B-тестирование версий моделей.
4. **Масштабирование**: Горизонтальное масштабирование через Kubernetes с readiness/liveness probes.
5. **Безопасность**: Валидация входных данных на уровне WAF, rate limiting, аудит логов.
