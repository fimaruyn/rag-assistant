# Ноутбуки проекта

В этой директории размещаются экспериментальные ноутбуки (Jupyter Notebook) для исследования данных, прототипирования моделей и проведения контролируемых экспериментов.

> **Важно**: Ноутбуки предназначены для **исследовательской работы**. Код, который становится частью продакшен-пайплайна (загрузка данных, предобработка, инференс), выносится в модули `src/` для обеспечения переиспользуемости, тестируемости и воспроизводимости.

---

## Доступные ноутбуки

| Файл | Описание | Статус |
|------|----------|--------|
| [`01_eda.ipynb`](01_eda.ipynb) | **Разведочный анализ данных (EDA)** для датасета *Give Me Some Credit*. Включает загрузку через `src.data.loader`, анализ распределений, пропусков, выбросов, корреляционный анализ и формулировку гипотез для предобработки. | ✅ Завершён |
| [`02_modeling.ipynb`](02_modeling.ipynb) | **Обучение и сравнение моделей**. Реализует пайплайн: сплит данных → предобработка через `src.features.preprocessing` → обучение LogisticRegression / RandomForest / LightGBM → оценка метрик (ROC-AUC, Gini, Avg Precision) → сохранение артефактов модели. | ✅ Завершён |

## Как запустить ноутбуки

### Требования
- Активированное виртуальное окружение (`uv` или `venv`)
- Установленные зависимости: `uv pip install -r requirements.txt`
- Датасет `cs-training.csv` в директории `data/raw/`

### Запуск через Jupyter Lab
```bash
cd project
uv run jupyter lab notebooks/
```

### Запуск через Jupyter Notebook
```bash
cd project
uv run jupyter notebook notebooks/
```

### Порядок выполнения
1. Откройте `01_eda.ipynb` и выполните ячейки последовательно (`Shift+Enter`).
2. После завершения EDA перейдите к `02_modeling.ipynb`.
3. Результаты экспериментов сохраняются в `artifacts/eda/` и `artifacts/modeling/`.

> **Примечание**: Ноутбуки используют модули из `src/`, поэтому запуск должен производиться из корня проекта (`project/`), чтобы путь `src/` корректно разрешался.

## Связь с модульным кодом (`src/`)

| Задача в ноутбуке | Реализация в `src/` |
|-------------------|---------------------|
| Загрузка и валидация данных | `src/data/loader.py`, `src/data/validator.py` |
| Предобработка признаков | `src/features/preprocessing.py` |
| Сохранение модели | `joblib.dump()` в `artifacts/modeling/models/` |
| Конфигурация путей | `src/service/config.py` (через `pydantic-settings`) |

**Рекомендация**: Если вы обнаружили полезную логику в ноутбуке, которую планируете использовать повторно:
1. Выделите её в функцию/класс в соответствующем модуле `src/`.
2. Добавьте юнит-тест в `tests/`.
3. Импортируйте готовый компонент обратно в ноутбук.

Это обеспечивает:
- ✅ Тестируемость кода
- ✅ Отсутствие дублирования
- ✅ Воспроизводимость экспериментов

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| `ModuleNotFoundError: No module named 'src'` | Запускайте ноутбук из корня `project/`, а не из `notebooks/`. Проверьте, что в первой ячейке есть код добавления корня в `sys.path` (если требуется). |
| Медленная загрузка данных | Убедитесь, что `cs-training.csv` не сжат и находится на локальном диске. |
| Ошибки валидации `pandera` | Проверьте, что версия `pandera` в `requirements.txt` совместима с импортами (`import pandera.pandas as pa` для ≥0.19). |
| Ноутбук «зависает» на визуализации | Уменьшите размер выборки для отладки: `df_sample = df.sample(n=10000, random_state=42)`. |

---

> **Совет**: Перед коммитом очищайте вывод ноутбука (`Cell → All Output → Clear`), если он занимает много места, **или** сохраняйте с выводом, если визуализации важны для отчёта. В данном проекте ноутбуки коммитятся **с выполненными выводами**, так как они являются частью документации эксперимента.
