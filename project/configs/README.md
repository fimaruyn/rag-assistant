# Конфигурационные файлы

В этой директории хранятся настройки сервиса кредитного скоринга и шаблоны переменных окружения. Вместо статических `.yaml`/`.json` файлов в проекте используется современный подход на базе `pydantic-settings`, что обеспечивает типизацию, валидацию на старте и безопасную работу с секретами.

## Структура
- `.env` — локальный файл с реальными значениями (игнорируется `.gitignore`).
- `.env.example` — шаблон с описанием всех необходимых переменных окружения.
- `README.md` — документация по управлению конфигурацией.

## Управление настройками
Параметры сервиса вынесены в переменные окружения и загружаются в `src/service/config.py`. Это позволяет:
- Не хардкодить пути, порты и флаги в коде.
- Легко менять конфигурацию для разных сред (local, Docker, CI/CD).
- Валидировать типы и обязательность параметров при инициализации приложения.
- Соответствовать требованиям курса к безопасной работе с секретами (`faq.md §2.4`).

## Переменные окружения

| Переменная | Описание | Значение по умолчанию |
|------------|----------|-----------------------|
| `HOST` | Хост для запуска Uvicorn | `0.0.0.0` |
| `PORT` | Порт API-сервиса | `8000` |
| `LOG_LEVEL` | Уровень логирования (`DEBUG`, `INFO`, `WARNING`, `ERROR`) | `INFO` |
| `APP_VERSION` | Версия сервиса для ответа `/health` и Swagger UI | `1.0.0` |
| `APP_TITLE` | Заголовок API в документации | `Credit Scoring API` |
| `MODEL_ARTIFACT_PATH` | Относительный путь к сериализованному пайплайну модели | `artifacts/modeling/models/scorer_pipeline_v1.0.joblib` |

## Использование

### Локальный запуск
1. Скопируйте шаблон в рабочую конфигурацию:
   ```bash
   cp configs/.env.example configs/.env
   ```
2. При необходимости измените значения в `configs/.env`.
3. Запустите сервис (переменные подхватятся автоматически через `pydantic-settings`):
   ```bash
   uv run uvicorn src.service.main:app --reload
   ```

### Запуск в Docker
При старте контейнера конфигурация передаётся через флаг `--env-file`:
```bash
docker run -d --name scorer-service -p 8000:8000 --env-file configs/.env credit-scorer
```
Внутри образа переменные доступны процессу Uvicorn без дополнительных скриптов загрузки.

## Безопасность и лучшие практики
- **Никогда не коммитьте `configs/.env`**. Реальные значения хранятся только локально или в защищённых хранилищах секретов CI/CD.
- Файл `.env.example` должен содержать только имена переменных и безопасные значения-заглушки.
- Все пути к артефактам указываются относительно корня проекта (`WORKDIR /app` в Docker).
- Для продакшена рекомендуется использовать orchestrator-specific secrets management (Kubernetes Secrets, AWS Secrets Manager, HashiCorp Vault), а не плоские `.env`-файлы.

## Добавление новых параметров
1. Добавьте переменную в `configs/.env.example` с поясняющим комментарием.
2. Опишите поле в классе `Settings` (`src/service/config.py`), указав тип Python и значение по умолчанию.
3. Используйте `settings.NEW_PARAM` в коде сервиса.
4. Обновите таблицу в этом `README.md` и локальный `.env`.
5. Убедитесь, что новая переменная не содержит чувствительных данных, либо поместите её в отдельный защищённый конфиг.
