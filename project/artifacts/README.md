# Артефакты проекта

В этой директории хранятся промежуточные и финальные результаты работы проекта, не относящиеся к исходному коду. Артефакты генерируются автоматически в процессе выполнения ноутбуков (`notebooks/`) и необходимы для:
*   Визуализации результатов разведочного анализа (EDA);
*   Фиксации метрик сравнения моделей;
*   Хранения сериализованной финальной модели для деплоя в Docker.

## Структура директории

```text
artifacts/
├── README.md                        # Этот файл
├── eda/                             # Результаты разведочного анализа данных
│   ├── data/                        # Табличные отчёты (CSV)
│   ├── figures/                     # Графики и визуализации (PNG)
│   └── eda_report.md                # Текстовый отчёт по EDA
└── modeling/                        # Результаты обучения и валидации моделей
    ├── data/                        # Метрики и сравнения (CSV)
    ├── figures/                     # Графики обучения (ROC, Feature Importance)
    └── models/                      # Сериализованные артефакты моделей (Joblib)
```

## Описание содержимого

### 1. Результаты EDA (`artifacts/eda/`)
Содержит результаты анализа датасета *Give Me Some Credit* до начала обучения.
*   **`data/`**: Сводные таблицы со статистикой (`feature_statistics.csv`), отчёты о пропусках (`missing_values_report.csv`) и корреляции признаков.
*   **`figures/`**: Визуализации распределения целевой переменной, гистограммы признаков, матрицы корреляций и boxplot-графики выбросов.
*   **`eda_report.md`**: Автоматически сгенерированный Markdown-отчёт с выводами и гипотезами для предобработки.

### 2. Результаты моделирования (`artifacts/modeling/`)
Содержит результаты экспериментов с моделями (Logistic Regression, Random Forest, LightGBM).
*   **`data/`**:
    *   `model_comparison_results.csv`: Итоговая таблица с метриками (ROC-AUC, Gini, Avg Precision) для всех моделей.
    *   `feature_importance_best_model.csv`: Веса важности признаков для лучшей модели.
*   **`figures/`**:
    *   `01_roc_curves.png`: Сравнение ROC-кривых всех моделей на тестовой выборке.
    *   `02_feature_importance.png`: Визуализация важности признаков LightGBM.
*   **`models/`**:
    *   `scorer_pipeline_v1.0.joblib`: **Финальный артефакт**. Сериализованный пайплайн `sklearn`, включающий трансформеры предобработки и модель LightGBM. Используется сервисом в `src/service/` и упаковывается в Docker-образ.

## ️ Важное примечание по хранению и воспроизводимости

1.  **Версионирование моделей**: Файл `models/scorer_pipeline_v1.0.joblib` является бинарным объектом. В рамках текущего проекта он включён в репозиторий для упрощения демонстрации и сборки Docker-образа.
2.  **Воспроизводимость**: Все артефакты (графики, отчёты и модели) могут быть сгенерированы заново путём последовательного запуска ноутбуков:
    ```bash
    # 1. Запуск EDA
    uv run jupyter nbconvert --execute notebooks/01_eda.ipynb --to notebook --inplace

    # 2. Запуск обучения и сохранения модели
    uv run jupyter nbconvert --execute notebooks/02_modeling.ipynb --to notebook --inplace
    ```
3.  **Большие файлы**: Если размер бинарных файлов моделей превысит разумные пределы (например, >100 МБ), рекомендуется использовать внешние хранилища (DVC, S3, Google Drive), а в репозитории оставлять только скрипты загрузки. В текущем проекте модель компактна (<5 МБ), поэтому хранится локально.
