# Данные проекта

В этой директории находятся данные, необходимые для обучения модели и работы сервиса кредитного скоринга.

## 1. Правила хранения данных

Согласно требованиям курса (`faq.md §2.1–2.3`) и политике приватности:

✅ **Допустимо хранить:**
- Инструкции по загрузке открытых датасетов.
- Небольшие примеры (sample) для проверки структуры.
- Обезличенные учебные выборки.

❌ **Запрещено хранить в репозитории:**
- Реальные персональные или конфиденциальные данные.
- Полные «сырые» файлы датасетов объёмом более 50 МБ.
- Файлы с секретами, токенами или учётными данными.

Для данного проекта исходные CSV-файлы исключены из контроля версий через `.gitignore`.

## 3. Описание датасета

| Параметр | Значение |
|----------|----------|
| **Название** | Give Me Some Credit |
| **Источник** | [Kaggle Competition](https://www.kaggle.com/c/GiveMeSomeCredit) |
| **Объём** | ~150 000 записей |
| **Признаки** | 10 числовых (финансовые показатели, история просрочек, демография) |
| **Целевая переменная** | `SeriousDlqin2yrs` ∈ {0, 1} (серьёзная просрочка 90+ дней в течение 2 лет) |
| **Тип данных** | Открытый, полностью обезличенный, учебный |
| **Лицензия** | Разрешено использование в некоммерческих и образовательных целях |

Датасет не содержит персональных идентификаторов (ФИО, паспортные данные, контакты). Все значения представлены в агрегированном или нормализованном виде, что соответствует требованиям `faq.md §2.2`.

## 4. Инструкция по загрузке

Для локального запуска проекта необходимо самостоятельно скачать исходные файлы:

1. Перейдите на страницу соревнования: https://www.kaggle.com/c/GiveMeSomeCredit
2. Примите правила соревнования (кнопка `Rules` → `Accept`).
3. В разделе `Data` скачайте архив или отдельные файлы:
   - `cs-training.csv` (обязательно)
   - `cs-test.csv`, `sampleEntry.csv`, `Data Dictionary.xls` (опционально)
4. Распакуйте файлы в директорию `data/raw/`.
5. **Не коммитьте** эти файлы в Git. Убедитесь, что в корне проекта присутствует правило:
   ```gitignore
   data/raw/*.csv
   data/raw/*.xls
   data/raw/*.xlsx
   ```

При корректной структуре `src/data/loader.py` автоматически проверит наличие файлов, валидирует схему через `pandera` и подготовит данные к предобработке.

---

## 5. Использование в проекте

Данные обрабатываются модульным пайплайном:

| Этап | Модуль | Описание |
|------|--------|----------|
| **Загрузка** | `src/data/loader.py` | Чтение CSV, нормализация имён колонок, фильтрация аномалий (возраст <18 или >120) |
| **Валидация** | `src/data/validator.py` | Проверка типов, диапазонов, наличия целевой переменной через декларативную схему `pandera` |
| **Предобработка** | `src/features/preprocessing.py` | Импутация пропусков, лог-трансформация skewed-признаков, агрегация коррелирующих счётчиков, масштабирование |

Ожидаемая схема входных данных зафиксирована в `src/data/validator.py` и соответствует столбцам `cs-training.csv`.

---

## 6. Ограничения и воспроизводимость

- **Объём**: Полный датасет занимает ~12 МБ в сжатом виде и ~45 МБ в разархивированном. Хранение в Git нецелесообразно и нарушает рекомендации курса.
- **Воспроизводимость**: Инструкции в `README.md` и `project/README.md` позволяют любому участнику курса восстановить рабочую среду за ≤5 минут.
- **Изменения**: Структура признаков и типы данных зафиксированы. При обновлении версии датасета на Kaggle необходимо сверить схему с `src/data/validator.py`.

Для вопросов по данным или структуре пайплайна обращайтесь к документации в `project/report.md` или к модулям в `src/data/`.
